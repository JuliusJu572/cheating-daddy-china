# 软件流程说明（客户端）

更新时间：2026-03-04 06:22:17

## 1. 目标与范围

本文说明桌面端（Electron）在实际使用中的核心流程，重点包括：

- License Key / 账号登录两种使用方式
- 简历与 JD 的解析和注入
- `Ctrl+L` 实时转写与回答链路
- 缓存与防串规则

## 2. 用户进入流程

### 2.1 仅 License Key

1. 用户在首页输入 License Key。
2. 客户端本地解密为 API Key，并测试模型连接。
3. 连接通过后可直接使用 AI、简历解析、JD 解析、ASR。
4. 不依赖后台账号体系即可运行。

### 2.2 登录账号（邮箱/License 登录账号）

1. 用户在登录页完成认证，拿到 `userAuthToken`。
2. 客户端进入“有账号态”，可访问后端用户接口（简历记录、会话记录等）。
3. 本地上下文缓存按当前登录用户隔离。

## 3. 简历与 JD 流程

### 3.1 简历上传与解析（设置页）

1. 用户上传本地简历（PDF/DOCX/TXT）。
2. 客户端提取文本并调用模型进行结构化解析，输出用于答题的简历摘要。
3. 同时提取 `asrHotwords`（术语热词）供 ASR 使用。
4. 简历摘要用于 LLM 上下文注入，热词用于 ASR 识别增强。

### 3.2 JD 输入与解析（设置页）

1. 用户粘贴目标岗位 JD 文本。
2. 客户端调用模型生成精炼 JD 摘要（限长，保留关键要求）。
3. JD 摘要用于 LLM 上下文注入，不直接整段送 ASR。

## 4. 会话初始化与回答流程

每次初始化模型会话时，会构造 `User-provided context`，核心内容可包含：

- `[候选人简历]`
- `[目标岗位JD]`
- `[后端历史上下文]`（如有）
- `[用户自定义指令]`（如有）

模型后续回答都会基于当前会话的这份上下文。

## 5. Ctrl+L 实时转写流程

1. 第一次 `Ctrl+L`：开始实时转写。
2. 过程中：音频分段送 ASR，实时更新转写文本。
3. 第二次 `Ctrl+L`：停止转写并提交转写内容给模型回答。

说明：

- 回答阶段会使用本会话已加载的简历/JD上下文。
- ASR 仅使用 `asrHotwords` 提升术语识别，不直接使用整段 JD。

## 6. 防呆规则（重点）

### 6.1 上下文生效时机

- 简历/JD 是在“会话初始化”时注入。
- 如果在会话进行中才修改简历/JD，建议重新开始会话再提问。

### 6.2 缓存规则

- 未登录账号：简历/JD 不做持久化缓存，仅本次运行有效；关闭应用后需重新上传/填写。
- 已登录账号：才会缓存，并按用户隔离。

### 6.3 防串规则

- 已登录状态下，简历/JD/热词读取与写入都按当前用户作用域隔离，避免交叉。
