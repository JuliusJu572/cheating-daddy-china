---

## 2. 为什么不需要再部署到服务器？


| 传统方式       | Sealos 方式   |
| ---------- | ----------- |
| 购买/租用服务器   | 无需单独服务器     |
| 安装操作系统、数据库 | 平台提供托管数据库   |
| 自行配置网络、安全  | 平台提供内网/公网访问 |
| 运维、备份、监控   | 平台统一管理      |


Sealos 提供的是**托管数据库服务**，数据库已在平台中创建并持续运行，等同于「数据库已经部署完成」。

---

## 3. 连接方式

### 内网连接（推荐）

- 在 Sealos 控制台 → 数据库 → `cheating-daddy-china` → **连接信息** 中查看
- 包含：Host、Port、用户名、密码、连接字符串
- 适用于：与数据库部署在同一 Sealos 集群内的应用

### 外网连接（可选）

- 如需从本地或其他网络访问，可在控制台开启「外网地址」
- 开启后平台会提供外网连接信息
